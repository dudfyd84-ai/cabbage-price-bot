# 에이전트로 작업 시작하기 (팀원용)

Claude Code(에이전트)에게 어떻게 명령해서 본인 이슈를 시작하는지 안내한다.
프로젝트 규칙은 `CLAUDE.md`에 있어 에이전트가 자동으로 따른다.

## 0. 준비 (한 번만)
1. Claude Code 설치 후 로그인.
2. 저장소 클론.
   ```
   git clone https://github.com/dudfyd84-ai/cabbage-price-bot.git
   cd cabbage-price-bot
   ```
3. 저장소 폴더 안에서 에이전트 실행(`claude` 또는 IDE 확장).

## 1. 첫 명령 (복붙 후 이슈 번호만 바꾸기)
에이전트에게 이렇게 지시한다.

> `ROADMAP.md`와 `CONTRIBUTING.md`, GitHub 이슈 #3을 읽고 내 작업을 파악해줘.
> 관련 파일을 살펴본 뒤 **구현 계획을 먼저** 보여줘. 내가 승인하면
> `feature/C-bom-manual-price` 브랜치를 만들고 구현해줘. 내 트랙(C) 파일 위주로만 수정하고,
> 데이터·모델 파일과 비밀키는 건드리지 마.

핵심 3가지만 지키면 된다 — **이슈 번호**, **계획 먼저**, **본인 브랜치**.

## 2. 구현 → 확인 → PR
- 계획을 검토하고 승인하면 에이전트가 구현한다.
- "로컬에서 `python app.py`로 돌려서 동작 확인하고 결과 보여줘"라고 지시한다.
- 다 되면 "커밋하고 push한 뒤 PR 올려줘. PR 본문에 `Closes #3` 넣어줘"라고 지시한다.
- **`main`에 직접 push는 막혀 있다.** 에이전트도 반드시 `feature/*` 브랜치로 PR을 올린다.

## 3. 리뷰 → 병합
- PR이 올라오면 팀리드가 리뷰·승인 → `main` 병합 → 자동 배포.
- 다음 이슈(2순위)로 넘어갈 때 1번부터 반복한다.

## 4. 트랙별 첫 프롬프트 예시
각자 본인 이슈 번호·브랜치로 시작한다.

- **B @ryong9797** — 이슈 #2
  > 이슈 #2와 `SUPABASE_SETUP.md`를 읽고 Supabase 계정화 구현 계획을 세워줘. 승인하면 `feature/B-supabase-auth` 브랜치에서 이메일 로그인부터 구현해줘.
- **C @kjw0106** — 이슈 #3
  > 이슈 #3을 읽고 BOM 수동 단가 입력 구현 계획을 세워줘. 승인하면 `feature/C-bom-manual-price` 브랜치에서 구현해줘.
- **D @clover1118-sudo** — 이슈 #4
  > 이슈 #4를 읽고 카카오 급등 알림톡 발송 모듈 구현 계획을 세워줘. 승인하면 `feature/D-kakao-alert` 브랜치에서 1품목 시범부터 구현해줘.
- **A 팀리드** — 이슈 #1
  > 이슈 #1을 읽고 예측 적중률 라이브 로깅 계획을 세워줘. 승인하면 `feature/A-predict-log` 브랜치에서 구현해줘.

## 5. 좋은 습관
- 에이전트가 낸 **계획을 먼저 검토**하고 승인한다(바로 코딩시키지 않기).
- 애매하면 "왜 이렇게 했어?"라고 되묻는다.
- 본인 트랙 밖 파일을 바꿔야 하면 담당자와 먼저 협의(PR 코멘트).
- 막히면 이슈에 상황을 코멘트로 남겨 팀과 공유한다.
